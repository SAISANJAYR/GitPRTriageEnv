# Evaluation & Results Directory 📊

This directory contains the entire history of the model's performance grading across the different stages of the Meta × Scaler OpenEnv Hackathon project. It is structured to allow judges to easily understand the progression from the baseline logic up to the final V2 Curriculum-Trained RL agent.

## Directory Structure

### `1. pre_training/`
Contains the `baseline_summary.json`. This data was generated by evaluating the completely untrained, zero-shot `Qwen2.5-1.5B-Instruct` base model against the PR Regression Audit environment. It serves as the bottom floor for our improvement metrics.

### `2. post_V1_training/`
Contains the `trained_summary.json` representing the **V1 Model** performance. 
- **Training**: 200 GRPO steps on an L4 GPU without Curriculum Learning.
- **Adapter**: `SaiSanjayR/pr-triage-grpo-adapter`
- **Results**: Shows massive improvements in the *Easy* difficulty, but struggled significantly on *Medium* and *Hard* due to a lack of structured progression during training.

### `3. post_V2_training/`
Contains the `trained_summary.json` representing the **V2 Model** performance.
- **Training**: 400 GRPO steps on an A10G/A100 GPU utilizing **Dynamic Curriculum Learning** and Anti-Reward Hacking Guards.
- **Adapter**: `rsd-06/pr-regression-audit-grpo-adapter-v2`
- **Results**: Demonstrates robust performance across all three difficulties, successfully solving deep integration defects that the V1 model hallucinated on.

### `4. checkpoints/`
This folder contains the actual, raw **LoRA Adapter Weights** downloaded directly from the Hugging Face Hub. 
- `v1_adapter/`: The `SaiSanjayR/pr-triage-grpo-adapter` weights.
- `v2_adapter/`: The `rsd-06/pr-regression-audit-grpo-adapter-v2` weights.
These are stored locally so judges can easily load them via `PeftModel.from_pretrained()` and run local inference without needing to download them over the network during the evaluation.

### `5. grpo_dataset/`
Contains the structured PR review data (`grpo_dataset.jsonl`) generated organically from the environment, used to train the models via reinforcement learning.

### `6. plots/`
Contains visualizations generated during data collection or evaluation comparing the learning curves.
